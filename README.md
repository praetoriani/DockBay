## Introduction to the DockBay Project – How it all began ...

When I started learning Linux by discovering WSL and HyperV, I also discovered Docker by mistake. I liked the concept of Docker and how it works. So, I also started working with Docker. I immediately set out to find useful containers that I could set up in Docker. After a while, I quickly realized that it makes a lot of sense to set up and manage Docker projects using Compose files. And that was the actual trigger that turned my initially rather harmless interest in Linux into this Docker project. So, rather than boring you any further with details about my personal background, let me tell you what this project is actually about.
<br/><br/>

To make it short:<br/>
The **🐳 DockBay Project** is the easy way to setup a whole Docker Environment (using Docker Community Edition, Compose Plugin and Buildx Plugin) including Traefik Reverse Proxy, Database Host (with MariaDB, PostgreSQL and MongoDB) and a full set of several App Containers (like Uptime Kume, Prunemate, DBgate, Dockman, Excalidraw, Passbolt, Sharkord, Planka, etc.).
